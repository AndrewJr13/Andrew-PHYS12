This was a very simple process. Using the LiquidCrystal constructor, I defined my rs, enable, D4, D5, D6, and D7 pins and the display was up and working just like that. <pre><code class="language-arduino"> LiquidCrystal lcd(8, 3, 4, 5, 6, 7); </code></pre> The only thing left to do was incorporate an analogWrite on pin 11 to adjust the constast of the display. On the actual LCD board, the pin named V0 is the contrast controller. In my case, I connected V0 to pin 11 on my board. After that the last thing to do was to begin the display! <pre><code class="language-arduino"> analogWrite(11, 140); lcd.begin(16, 2); </code></pre> Next, I decided to code the sensor and lock before I assembled the project. I already had the electronics mounted and running so I started the coding process. First part of coding was getting the sensor the recognize gestures. Unfortunatley the Adafruit library that went with the sensor was not able to be used since it collided with the LiquidCrystal library. Even more of a drawback was the fact that the only alternate library (SparkFun_APDS9960) was a bit more complicated. Mainly, the fact that the Adafruit library was able to use if statements for the gestures. For example: <br> <pre><code class="language-arduino"> if (DIR_RIGHT) { //Do this } </code></pre> <br> Unfortunatley the Sparkfun library would not work with if statments. This was mainly due to the readGesture() method. This method was very confusing. After further research, I found out that the method returned an integer value. I soon was excited and thought I could incorporate an if statement with the "==" binary operator if each direction returned a different integer. Unfortunatley this was not the case. After further testing, I found the method returned 0 for every direction. Something that still confuses me to this day. So my last resort was to take a peek at the example code for the gesture sensor and found out I would have to use a switch and cases. Switches and cases were something I was familiar with from my Java and Javascript experience but I never really loved them. Unfortunatley it seemed this was the only way. The next step was to incorporate code for an interrupt. First I bound the the interrupt to pin 2 on my Arduino, and set an interrupt flag. I also created my sensor object which i just called "apds." <pre><code class="language-arduino"> #define APDS9960_INT 2 int isr_flag = 0; SparkFun_APDS9960 apds = SparkFun_APDS9960(); pinMode(APDS9960_INT, INPUT); </code></pre> Next I used Arduino's "attachInterrupt" method and a method that changes the interrupt flag called "interruptRoutine." <pre><code class="language-arduino"> attachInterrupt(0, interruptRoutine, FALLING); void interruptRoutine() { isr_flag = 1; } </code></pre> Next it was time to create the loop and incorporate the interrupt inside the loop. First, the loop checks if isr_flag = 1 which it should through the interruptRoutine method which sets it to 1. Next, the program detaches the Interrupt and resets isr_flag. I then implemented my main sequence there (I will get to that later), reset isr_flag, and then re-attached the interrupt. This sequence of code would mean that everytime I waved my hand in front of the sensor, my loop wold run my gestureSequence() method which I will showcase later. <pre><code class="language-arduino"> if ( isr_flag == 1 ) { detachInterrupt(0); lcd.clear(); gestureSequence(); isr_flag = 0; attachInterrupt(0, interruptRoutine, FALLING); </code></pre> After creating the interrupt, I started working on the gestureSequence() method. First off was an if statement using a method from the Sparkfun library. Very simple: <pre><code class="language-arduino"> if ( apds.isGestureAvailable() ) { </code></pre> Next I created my switch. In my case, the switch will keep running the readGesture() method until it matches it to a specific case. In each case, I cleared the LCD, set the cursor so the text would be centered then printed the direction corresponding to the gesture. For example, here is what the DIR_UP case looked like. <pre><code class="language-arduino"> lcd.clear(); cd.setCursor(7, 0); lcd.print("up"); delay(500); break; </code></pre> After this, my next step was to set up my servo and test it with just one gesture. Here's the simple setup code using the Servo library. <pre><code class="language-arduino"> const int MTR_PIN = 12; Servo lock; lock.attach(MTR_PIN); lock.write(29); //starts in locked position when code is uploaded </code></pre> As I said before, I ran the servo lock with just one gesture. Here the code and a quick video. <pre><code class="language-arduino"> lcd.clear(); lcd.setCursor(6, 0); lcd.print("left"); lock.write(129); delay(500); lcd.print("welcome"); break; </code></pre> <br> <video controls width="1100" height="700"> <source src="left.mp4" type="video/mp4"> </video> Next step was the most time consuming but also the most important. I took some time to brainstorm a way to have a sequence of hand gestures activate the servo and unlock the door. Ultimatley, I decided to use a simple integer array. I started by declaring two arrays. One for the current gesture sequence and the other would be the correct sequence. <pre><code class="language-arduino"> int current[] = {0, 0, 0, 0, 0, 0, 0}; int correct[] = {1, 3, 4, 3, 2, 4, 3}; //each number is a gesture 1 = up, 2 = down, 3 = left, 4 = right </code></pre> Next course of action was having each case set an element of the array to an integer that matches the direction. I defined each number above in the code. So 1 is up, 2 is down, 3 is left, 4 is right. Simple enough. Now im not very familiar with C so I originally thought of arrayLists. Whenever I code with Java, I love using arrayLists for things like these. It would be much easier to use the add and remove methods inside the cases but unfortunatley it didn't seem that something like this would be possible on C. So I created an integer called "index." Index would increment evertime a gesture was detected and then my code would set that index of the array to an integer based off of the gesture. For example, if index = 0 and the user gestures downward, index 0 of the "current" array will be set to 2 and index will increment so that the next gesture will occupy the next index of the array. So now the DIR_UP case was looking something like this. <pre><code class="language-arduino"> case DIR_UP: lcd.clear(); lcd.setCursor(7, 0); lcd.print("up"); delay(500); current[index] = 1; index++; count++; //see next section </code></pre> So now I had the arrays taken care of. I now needed to compare both arrays. If the arrays matched, the servo would open and if not then it would stay closed. Fortunatley, my prior coding experience came in handy. First I needed a way for the program to know when the amount of gestures performed matched the number of elements in the "correct" array so the sensor would not run past that point. So I created a "count" variable. "Count" would increment inside of every case and I ran a quick if statement to check when count was the same as the size of the correct array. Next I created a for loop that would run through both arrays and compare each element inside. If the elements matched, a variable called "numRight" would increment. <pre><code class="language-arduino"> if (count == sizeof(correct)) { for (int i = 0; i < sizeof(correct); i++) { if (current[i] == correct[i]) { numRight++; } } </code></pre> After this, it was a simple journey to the end. I ran an if statement that checked if numRight was equal to the size of the correct array. If it is, that means every element of the "current" array matches every element of the "correct" array. After this I cleared the LCD, printed "welcome," and turned the servo. I also added two loops inside this statment. One was simply to control my LED and the other reset the elements of the "current" array. I then set index, count, and numRight back to 0. <pre><code class="language-arduino"> if (numRight == sizeof(correct)) { lcd.clear(); lock.write(129); lcd.print("welcome"); for (int i = 0; i < 3; i++) { analogWrite(greenLight, 255); delay(100); analogWrite(greenLight, 0); delay(100); } delay(1500); lcd.clear(); for (int i = 0; i < sizeof(current); i++) { current[i] = 0; } numRight = 0; count = 0; index = 0; } </code></pre> After this, I created another if statement for an incorrect sequence. All it does is flash the LED red, print "incorrect," then resets the "current" array and sets index, count, and numRight back to 0. <pre><code class="language-arduino"> if (numRight != 7 && count != 0) { lcd.clear(); lcd.print("incorrect"); analogWrite(redLight, 255); delay(2000); analogWrite(redLight, 0); lcd.clear(); for (int i = 0; i < 7; i++) { current[i] = 0; } numRight = 0; count = 0; index = 0; } } //end if (count == sizeof(correct)) </code></pre> Finally, here is a video of the first sequence I tested. It contained 4 gestures and to my suprise, worked the first try. <br> <video controls width="1100" height="700"> <source src="sequence.mp4" type="video/mp4"> </video> <br> The last part of my program is the proximity lock. Fortunatley, this too could be done in a case. My DIR_FAR case detected when a user would slowly distance themselves from the sensor and I planned to have it lock the door. First, I reset count, index, and numRight to 0. Then I clear the LCD and print "locking..." I then use a loop to flash the LED blue 2 times. Then the servo locks and the "current" array is reset. Thats it! <pre><code class="language-arduino"> count = 0; index = 0; lcd.clear(); lcd.print("locking..."); delay(1000); for (int i = 0; i < 2; i++) { analogWrite(blueLight, 255); delay(100); analogWrite(blueLight, 0); delay(100); } lock.write(29); lcd.clear(); numRight = 0; for (int i = 0; i < 7; i++) { current[i] = 0; } break; </code></pre> Finally, here is the entire program. <br> <pre><code class="language-arduino"> #include <Wire.h> #include <SparkFun_APDS9960.h> #include <LiquidCrystal.h> #include <Servo.h> LiquidCrystal lcd(8, 3, 4, 5, 6, 7); delay(1000); lcd.clear(); break; } } } </code></pre> <br> <br> <br> <head>Final steps: Assembly</head> <br> Now that I had finished my code, it was time for part making and assembly.
